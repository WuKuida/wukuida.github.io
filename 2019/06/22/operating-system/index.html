1.2 什么是操作系统

就是一软件

  • 管理应用
  • 管理硬件资源

  • 虚拟：让每个用户都觉得有一个计算机为他服务
  • 异步：程序可以不是一直运行，可以中断，但是在相同的运行环境下，结果一定要相同

2.2 操作系统的系统调用，异常，中断

系统调用：应用程序主动向操作系统发出服务请求
异常：（来源于不良的应用程序）非法指令或者其它坏的处理状态，使得操作系统不得不进行处理
中断：来自不同的硬件设备的计时器和网络的中断

中断处理过程

  1. 保存当前处理状态
  2. 中断服务程序处理
  3. 清楚中断标记（由硬件产生）
  4. 恢复之前保存的处理状态

异常处理流程

  1. 保存现场
  2. 异常处理
    • 处理方式一：杀死产生异常的服务
    • 处理方式二：修复产生异常的原因（如果这个原因是操作系统造成的话），重新执行
  3. 恢复现场

系统调用处理流程：

以标准C库为例

  1. 应用程序调用printf（）时
  2. 直接或间接的通过lib库调用write（），
  3. 此时会cpu从用户态到内核态，应用程序堆栈切换到内核堆栈
  4. 在执行write的时候，操作系统可能会访问相应的硬件，比如屏幕
  5. 系统调用处理结束返回成功或失败
  6. 应用程序获取系统调用的结果之后，继续执行（当然应用程序可能调用之后就去执行别的了，不用等系统调用返回结果）

3.1 计算机体系结构/内存分层体系

操作系统对内存管理：

  • 抽象：逻辑地址空间，把实际的地址空间抽象成连续的逻辑地址空间
  • 保护：应用程序之间独立的地址空间
  • 共享：可以访问的相同内存空间
  • 虚拟化：更多的地址空间（虚拟内存）

3.3 连续内存分配：内存碎片与分区动态分配

内存碎片：

  • 外部碎片：在分配单元间的未使用内存
  • 内存碎片：在分配单元中的未使用内存

内存分配算法：

  • 首次适配算法：按地址从低到高找到第一块满足需求的内存块
    • 简单（优点）
    • 易于保留较大的地址空间（优点）
    • 容易产生外部碎片（缺点）
    • 不确定性（缺点）
  • 最佳适配：找到一个刚好大于需求的空闲内存块
    • 最小化外部碎片产生的尺寸（优点）
    • 比较适合较小需求的内存块分配（优点）
    • 简单（优点）
    • 容易产生外部碎片（缺点）
    • 分配慢（缺点）
    • 容易产生的过小的外部碎片，不好用（缺点）
  • 最差适配：找到最大的一个空闲内存块来满足需求
    • 比较适合中等尺寸需求的内存块分配（优点）
    • 重分配慢（缺点）
    • 较多外部碎片（缺点）
    • 会破坏大块的，容易导致大块的需求不能满足（缺点）

3.4 连续内存分配：压缩式与交换式碎片整理

压缩式：
将分配的内存压缩到一起，去掉中间的碎片空间
注意点：

  • 要在程序停止的时候做这个
  • 要注意内存拷贝的开销

交换式：
将内存中应用程序（数据）暂时放到硬盘（虚拟内存）中
注意点：

  • 要选择哪个程序放到硬盘（虚拟内存）中
  • 要在什么时候进行这个操作

4.0 非连续分配的优缺点

非连续分配的优点：

  • 一个程序的物理地址空间是非连续的
  • 更好的内存利用和管理，减少碎片
  • 允许共享代码和数据（共享库等…)
  • 支持动态加载和动态链接（不懂）

非连续分配的缺点：

  • 如何建立虚拟内存和物理地址之间的转换

4.1 非连续内存分配：分段

  • 可以按一下几个模块分段

    1 堆

    2 栈

    3 程序数据

    4 程序text段（包括库和用户代码）

  • 段地址访问机制，程序访问内存地址需要一个二维的二元组（s，addr）

    1 s是段地址

    2 addr是段内偏移

  • 通过存储在cpu内MMU的段表（seg table),由操作系统维护

    1 逻辑地址的s，就是段表的index

    2 段表那存了每段的起始物理地址base，和每段的长度limit，如果逻辑地址的段内偏移大于该段的长度，cpu就会产生内存异常报错

4.2 非连续内存分配：分页

基本概念：

  • 划分物理内存至大小固定的帧（块）frame
  • 划分逻辑内存至大小固定的页 page
  • 建立逻辑地址到物理地址的转换
    • 页表（由操作系统建立，存在内存中）
    • MMU中的快表（TLB）

逻辑地址到物理地址映射过程：

  1. 对于一个逻辑地址（page_num, offset)
  2. 通过页表（实际就是一维数组，它的页表基址寄存器（PTBR）是操作系统设置的），将page_num转化成frame_num(frame_num = tabel[page_num+PTBR])
  3. 得到（frame_num, offset)
  4. frame_num * 帧（块，页）大小 + offset 就是物理地址

4.3 非连续内存分配：页表-概述

每个运行的程序都会有一个页表，属于程序运行状态，会动态变化

页表的结构：

  • PTBR:页表基址寄存器
  • table[page_num + PTBR]并不只有frame_num
    • Flags(标识位,前面的几个bit），是否写过等
      • dirty bit
      • resident bit（判断是否存在相应物理地址）
      • clock/reference bit
    • frame_num:帧号

时间问题：因为页表空间比较大，所以放到内存中，所以逻辑地址到物理地址需要访问两次内存。
解决方案：通过cpu中的MMU的TLB快表缓存（Caching）

- TLB使用associative memory（关联内存），实际上就是map，(key, value) = (page_num, frame_num)
- 如果TLB命中，逻辑页号对应的物理页号可以直接获取
- 如果TLB没有命中，将对应的映射关系存入TLB


空间问题：对于64位的操作系统，如果帧的大小是1KB，那么页表（数组）的长度应该为$2^{64-10}$,如果每个应用都有对应的页表，那就更大了
解决方案：通过间接访问（多级也表）4.4 页表-多级

- 将page_num分成page_num1和page_num2
- 将page_num1通过一级也表，找到�二级也表的基址（base),然后将page_num2映射到物理帧号
- 因为访问一级页表里面，可能就不存在对应的二级页表，所以节省了空间

5.1 虚拟内存的起因

内存不够用
最理想的内存，容量大，速度快，能保存数据

5.2 覆盖技术

目标：
在较小的内存中运行较大的程序。

实现：
按程序划分成若干个相对独立的模块，让那些不会同时执行的模块共享同一块内存区域。

  • 必要部分（常用功能）的代码和数据要常驻内存
  • 可选部分（不常用功能），平时放在外存（磁盘）中，需要时加载到内存
  • 不存在调用关系的模块不必同时存在内存中，故他们可以共享同一块内存空间

优点：
可以在相对比较小的内存中跑比较大的程序

缺点：

  • 太麻烦了，要程序模块间的覆盖要程序员自己设计
  • 从外存（磁盘）加载到内存，费时

5.2 交换技术

目标：
多道程序在内存中的时候，让需要运行的程序或将要运行的程序获得更多的内存资源

实现：

  • 将暂时不用的程序从内存移动到外存中（swap out)
  • 将需要运行但是放在外存中的程序移入到内存中（swap in)

考虑的问题：

  • 需要考虑换入换出的时间（只有当有程序内存不够用时才进行）
  • 交换区的大小（感觉就是虚拟内存的概念）极端情况只有一个程序留在内存里，别的都放到交换区
  • 换出的程序，换入之后可能不在原来的地址空间

和覆盖技术的区别：

  • 覆盖技术

    • 颗粒度是程序内的一个模块
    • 需要程序员自己设计（太麻烦了）
    • 对正在运行中的程序进行操作

  • 交换技术

    • 颗粒度是一整个程序（太浪费时间了）
    • 由操作系统完成
    • 对没有正在运行的程序操作

5.3 虚存技术（上）

目标：

  • 像覆盖技术那样，只把程序的部分内容放在内存中，�且无需人为干预
  • 像交换技术那样，能够实现进程在内存与外存之间的交换

程序的局部性原理：

  • 时间局部性：一条指令（数据）一次访问和下一次访问都集中在较短的时间
  • 空间局部性：当前指令（数据）和下一条指令（数据）都集中再较小的一块区域

基本特征：

  • 大的用户空间：因为把磁盘也当成内存使用
  • 部分交换：与交换技术相比，虚拟存储的换入和换出的内存是对部分虚拟地址空间进行的
  • 不连续性：物理内存分配的不连续，虚拟地址空间使用的不连续

5.4 虚存技术（下）

虚拟页式内存管理：

  • 大部分虚拟存储系统都采用虚拟页式存储管理结构：即在页面存储管理的基础上，增加请求调页和页面置换功能
  • 基本思路：
    • 将程序载入到内存中的时候，只将该程序必要的一部分先载入内存中
    • 在运行过程中，如果发现要运行的程序或者需要访问的数据不在内存中，则向操作系统发出缺页中断请求，让操作系统将外存中的页面载入内存

为了让有些页面常驻内存还有锁定位（lock bit)，这些页面不在页面置换算法的处理范围内。

缺页中断处理流程：

  1. 如果内存中间中有空闲的物理页面，则分配物理页面f，然后跳到4
  2. 采用某分页置换算法，找到需要被替换的物理页f，该物理页对应的逻辑页是p。如果该页被修改过则将它写回到外存。
  3. 将p所对应的页表项进行修改，驻留位改成0
  4. 将需要访问的页面p2载入到物理页f中
  5. 将p2对应的也表项的驻留位改成1，无力页号改成f
  6. 重新运行被中断的指令

如果是运行过程中产生的数据，在外存中本来页没有，这种是怎么处理的？？？因为我感觉在内存新产生的数据再外存中是没有的，流程上应该稍微有一点不一样吧？？？

后备存储（Backing Store）：
如何保存未被映射的页：

  • 能够简单地识别在�二级存储器中的页
  • 交换空间（磁盘或者文件）：特殊格式，用于存储未被映射的页面

后备存储（backing store)概念：

  • 一个虚拟地址空间的页面可以被映射到一个文件（在二级存储中的某个位置）–数据文件
  • 代码段：映射到可执行的二进制文件
  • 动态加载的共享库程序段：映射到动态调用的库文件
  • 其他段：可能被映射到交换文件（swap file)–运行过程中产生的数据

6.1 最优页面置换算法

功能：
当缺页中断产生的时候，需要调入新的页面但是内存已满，需要选择内存中换出的页面

目标：
尽量减少缺页中断的次数

最优页面置换算法：
当一个缺页中断产生的时候，找到内存中离下一次访问最久的那个页面，把它替换出去（最理想的，不可能实现）

6.2 先进先出页面置换算法（FIFO：First-in First-out）

基本思路：

选择内存中驻留时间最长的页面并淘汰掉，相当于一个队列（链表），先进先出。

优点：

只需要一个指针

缺点：

忽略了程序运行的时空局部性，经常调出经常访问到的页，并且有Belady现象

6.3 最近最久未使用算法（LRU:Least Recently Used)

基本思路：

选择内存中最久未被使用的那个页面淘汰

优点：

相对比较接近最优页面置换算法，也利用了程序运行的时空局部性

缺点：

因为需要记录每个页面的使用时间的先后次序。假如用的是链表：需要用到两个指针。如果访问了在内存中存在的页面，则需要在链表中找到这项把这一项从链表中移到链表的头部。如果不存在，则淘汰链表尾部的元素，将新加入的放入链表头

6.4 时钟页面置换算法（Colock）

基本思路：

  • 如果一个物理页面最近被访问了（读/写），那么在页表中对应的那个页表项内的访问位（access bit)会被设置成1

  • 将存在内存中的页面的页表项组成一个环形链表（类似一个时钟）

  • 当发生缺页中断的时候，判断当前指针指向的页表项中的访问位是否位1，是1的话设置成0，指针往后挪一位。最先找到一个访问位是0的，则就是我们需要调出去的页面

  • 找到需要调出内存中的页之后，还是一样的操作，将该页表项中的驻留位设置成0，如果这个页被写过，还需要同步到外存中。然后将我们要加入的逻辑页对应的页表项的物理地址改成刚刚腾出来的，外存的数据写到该物理页，驻留位改成1，访问位设置成1。

  • 指针往后挪一个,不挪的话，在极端情况下刚刚加入的时候，恰巧其它页表项的访问位也都是1，那么下一次还是先淘汰了这个新加入的页，感觉不太合理了。
