1.2 什么是操作系统

就是一软件

  • 管理应用
  • 管理硬件资源

Kernel-操作系统内部组件

包括：

  • CPU调度器
  • 物理内存管理
  • 虚拟内存管理
  • 文件系统管理
  • 中断处理与设备驱动

特点：

  • 并发：一段时间内有多个程序在运行
  • 并行：一个时间点同时存在多个运行的程序
  • 共享：”同时”访问
  • 虚拟：让每个用户都觉得有一个计算机为他服务
  • 异步：程序可以不是一直运行，可以中断，但是在相同的运行环境下，结果一定要相同

2.2 操作系统的系统调用，异常，中断

系统调用：应用程序主动向操作系统发出服务请求
异常：（来源于不良的应用程序）非法指令或者其它坏的处理状态，使得操作系统不得不进行处理
中断：来自不同的硬件设备的计时器和网络的中断

中断处理过程

  1. 保存当前处理状态
  2. 中断服务程序处理
  3. 清楚中断标记（由硬件产生）
  4. 恢复之前保存的处理状态

异常处理流程

  1. 保存现场
  2. 异常处理
    • 处理方式一：杀死产生异常的服务
    • 处理方式二：修复产生异常的原因（如果这个原因是操作系统造成的话），重新执行
  3. 恢复现场

系统调用处理流程：

以标准C库为例

  1. 应用程序调用printf（）时
  2. 直接或间接的通过lib库调用write（），
  3. 此时会cpu从用户态到内核态，应用程序堆栈切换到内核堆栈
  4. 在执行write的时候，操作系统可能会访问相应的硬件，比如屏幕
  5. 系统调用处理结束返回成功或失败
  6. 应用程序获取系统调用的结果之后，继续执行（当然应用程序可能调用之后就去执行别的了，不用等系统调用返回结果）

3.1 计算机体系结构/内存分层体系


操作系统对内存管理：

  • 抽象：逻辑地址空间，把实际的地址空间抽象成连续的逻辑地址空间
  • 保护：应用程序之间独立的地址空间
  • 共享：可以访问的相同内存空间
  • 虚拟化：更多的地址空间（虚拟内存）
